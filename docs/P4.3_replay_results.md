# P4.3 replay results: affordance minting from real ls20 / tu93 recordings

**What ran:** `replay_affordance` (src/newhorse/redux_arch/replay.py) drives the calibrated
`AffordanceMintBridge` over recorded runs offline (no SDK). A pre-pass learns the perception basis from the
recording's own motion (cursor = smallest translating colour; per-action vec = mode of its non-zero shifts); the
main pass calibrates the leak-free passable set (colours the cursor stepped onto, frozen after warmup) and mints
by two-part MDL on the held-out steps. LAW 0 / RULE 0.7: multiple recordings per game, pixels/numbers decide.

## Results (max_size=1, warmup=20, min_exceptions=15, min_hits=2)

**ls20 — SOLID.** cursor=12, passable={3} (green), vecs clean ±5 on all four actions. On every recording with a
real moved/blocked mix (≈70 moved / ≈44 blocked) the minter mints the **colour-agnostic INTENDED_FREE** — and
*only* INTENDED_FREE (6/8 recordings). The two pure-residual runs (109 moved / 1 blocked; and 0 moved / 243
blocked) correctly **NO-MINT** — a pure exception stream has no structure to compress, so the guard declines.
This is the intended behaviour and is now pinned by `tests/test_replay_real.py`.

**tu93 — partial.** cursor=4 (the yellow dot) is now correctly identified; the retracted grey mint is gone. On
blocking-rich recordings it mints **INTENDED_COLOUR==0** — colour 0 (black corridors) is the *correct* passable
colour (matches the LAW-0 pixel finding). BUT it mints the colour-SPECIFIC atom, not INTENDED_FREE, because its
passable set comes out **{0, 5}** (contaminated with grey): the tu93 per-action vecs are noisy (`(7,-1)`,
`(1,7)` cross-axis instead of a clean single-axis stride), so the projected "cell ahead" occasionally lands on
the grey surround on a step scored as `moved`, mis-crediting grey(5) as floor. With 5 in the passable set,
INTENDED_FREE is impure and the MDL prefers the accurate INTENDED_COLOUR==0.

## Honest boundary (do NOT overclaim)
The live cross-GAME echo (ls20 + tu93 both → INTENDED_FREE → PROMOTE) does **not** fire yet, because the two
games mint DIFFERENT predicates: ls20→INTENDED_FREE, tu93→INTENDED_COLOUR==0. fail→mint is now solidly real on
ls20's own frames (the colour-agnostic, transferable predicate); the ECHO across the two real games awaits a
clean tu93 passable set.

## RESOLVED — the live cross-game join now fires (head 165/165)
Two fixes closed it:
1. **Axis-snapped vecs** (`replay._dominant_axis_vec`): each per-action displacement is snapped to its dominant
   axis (sign × median magnitude), removing the cross-axis jitter (`(7,-1)` → `(7,0)`).
2. **Calibrate from the ACTUAL trajectory, not the projection** (`bridge._entered_colour`): passability is now
   learned from the before-state colour of the cell the cursor TRULY entered (its after-position), not the
   projected cell-ahead. The projection overshoots into the grey surround near boundaries (tu93: 4/31 moved steps
   projected into grey, but the actual-entered colour was black 0 on ALL of them). This drops grey from tu93's
   passable set: passable → {0}.

With passable clean, **both ls20 (passable {3}) and tu93 (passable {0}) mint the colour-agnostic INTENDED_FREE**
from their own recorded residuals. `tests/test_replay_promote.py` then runs the full join on real frames:
INTENDED_FREE minted on ls20 → HELD → echoed on tu93 (a DIFFERENT game/palette) → **PROMOTED into Γ** → Γ
**explains a third held-out maze residual by MDL WITHOUT re-minting** (~95–105 bits saved). `explains()` was
aligned to the minter's two-part-MDL criterion (compresses), not perfect purity, since real residuals are noisy.

This is Region III non-empty end-to-end on REAL ARC frames at the single-predicate level: the first predicate the
navigation kernel lacked, minted AND consolidated from actual play across two games.
